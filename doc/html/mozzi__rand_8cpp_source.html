94 in Mozzi's rand() function. This can be useful if you want random sequences to
95 be different on each run of a sketch, by seeding with a fairly random input.
96 randSeed() called without a parameter uses noise from reading the Arduino's
97 internal temperature as the seed, a technique discussed at
98 http://arduino.cc/forum/index.php/topic,38091.0.html, borrowing code put there
99 by Rob Tillaart.
100 @note IMPORTANT!!! Make sure you set analogReference() to suit your sketch after calling randSeed().
101 randSeed() sets analogReference to DEFAULT. Sorry about that.
102 @note This is quite slow, so best to call it only in setup().
103 @note It's not perfect, as discussed in the forum thread, but probably a pretty good option.
104 @note It might only work with some processors: (from the thread)
105 "...ATmega328P in DIP, possibly others but the duemilanove and uno will do it at least."
106 So far, gizduino's __AVR_ATmega644P__ chip doesn't like it, so we use (long)analogRead(0)*analogRead(1) for that instead.
