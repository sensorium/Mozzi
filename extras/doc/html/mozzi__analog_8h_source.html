229 /*
230 Currently, if audio input is enabled it uses analog input channel 0 and doesn't
231 allow other channels to be used, to avoid interference between the strict audio
232 rate sampling and any control analog conversions which could be in progress when
233 the audio needs to be sampled. This might be solved in future, but workable
234 solutions for control inputs include RC circuits on digital pins, switches, or
235 serial connections (which also may require extra solutions for asynchronous
236 operation). There is a fork of Mozzi on github (posted on the mozzi-users list)
237 which includes a 2-wire async serial adaptation. It would be good eventually to
238 merge that with the main repo.
239 */
